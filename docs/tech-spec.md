# Win-Top 技术规范（逐步完善）

> 本文档聚焦技术架构与实现规范，供开发执行与后续细化。

## 1. 总体架构

- **桌面框架**：Tauri（Rust + Web 前端）。
- **核心服务**：Rust 调用 Windows API（WMI/ETW/Performance Counters/WinAPI）。
- **前端**：React 或 Vue + Fluent UI / Ant Design + Tailwind。
- **AI 层**：本地轻量推理 + 云端 API（复杂分析）。

```
UI Frontend (React/Vue)
        │ Tauri Bridge
Rust Core Service
        │ Windows APIs
WMI / ETW / WinAPI
```

## 3. 核心模块规范

### 3.1 资源监控模块
- **采样周期**：默认 1s，可配置。
- **数据缓存**：内存环形缓存保存最近 N 分钟数据。
- **输出格式**：统一 JSON 数据结构供前端消费。

### 3.2 进程与服务管理模块
- **进程枚举**：WinAPI / WMI。
- **权限要求**：涉及操作需管理员权限。
- **日志**：记录操作时间、目标 PID、执行结果。

### 3.3 网络与端口模块
- **端口解析**：结合 netstat 与 WMI 进行 PID 关联。
- **异常识别**：支持标记高频连接与陌生 IP。

### 3.4 脚本执行模块
- **脚本模板结构**：
  - name / description / params / risk / command
- **执行流程**：
  1) 用户确认
  2) 执行
  3) 输出日志
  4) 结果归档

### 3.5 AI 助手模块
- **输入**：系统状态摘要 + 用户问题。
- **输出**：解释、建议、脚本草案。
- **安全**：执行前必须人工确认。

---

## 4. 质量与安全规范

- **危险操作强制二次确认**。
- **所有执行命令需显示原文**。
- **日志必须可追溯**（导出 JSON/CSV）。
- **隐私**：敏感信息不得上传云端（默认本地）。
