# Win-Top

Windows Network、Disk、App Management

Win-Top 是一款专业、现代、美观的 Windows 资源管理工具，聚焦系统与应用资源的可视化监控与一键化管理，集成 AI 能力帮助用户理解系统状态并做出高效决策。它将网络、端口、进程、服务、磁盘、CPU、内存等核心资源统一为一个可扩展的桌面端工作台，并提供“脚本固化 + 可视化工具”能力，让复杂命令变成可理解、可重复的一键操作。

---

## 1. 产品定位与目标用户

**产品定位**
- **一体化 Windows 资源管理工作台**：统一监控、诊断、优化与执行。
- **面向非技术用户的可视化工具**：复杂命令可视化解释、可追溯。
- **面向开发者的效率工具**：端口管理、进程与服务治理、开发环境诊断。

**目标用户**
- **Windows 开发者**：需要高效管理服务、端口、进程与依赖。
- **电脑小白用户**：需要一键化“健康检查 + 修复建议”。
- **职场/办公人士**：需要提升系统效率、排查问题、管理常用工具。

## 2. 需求分析与功能定义（PM 视角）

### 2.1 核心目标
- **可视化**：将系统指标与复杂命令以图形化方式呈现。
- **一键化**：将高门槛命令沉淀为安全的可视化操作卡片。
- **智能化**：通过 AI 做解释、建议与风险提示。
- **可扩展**：工具与脚本能力可无限演进。

### 2.2 核心功能模块

#### A. 资源监控中心
- 实时看板：CPU / 内存 / 磁盘 / 网络。
- 指标历史趋势与异常波动提示。
- 资源分布：进程/应用资源贡献排行。

#### B. 进程与服务管理
- 进程树、PID、路径、签名、资源占用。
- 服务启停、依赖关系、启动项与权限。
- 快捷操作：结束进程、设置优先级、释放端口。

#### C. 网络与端口管理
- 监听端口、连接详情、关联进程。
- IP/端口异常识别（高频/陌生连接）。
- 一键防火墙策略生成与阻断。

#### D. 磁盘与存储管理
- 分区占用、热文件、大文件与冗余扫描。
- 清理建议、释放空间估算与风险提示。

#### E. 命令与脚本工具箱
- 预置常用命令（ipconfig、netstat、sfc、dism 等）。
- 可视化配置：命令参数可视化、可模板化。
- 执行记录：日志留存、结果对比。

#### F. AI 助手（Win-AI Core）
- **状态解释**：当前系统状态与风险解释。
- **建议优化**：基于系统状态提供操作建议。
- **脚本生成**：根据自然语言生成可执行脚本。
- **可视化确认**：执行前解释 + 风险提示。

### 2.3 需求优先级（MVP → 进阶）
- **P0（必须）**：资源监控、进程管理、端口查询、脚本工具箱、基础 AI 问答。
- **P1（重要）**：磁盘清理建议、异常检测、操作审计、脚本参数化 UI。
- **P2（增强）**：插件生态、脚本分享、跨设备管理、自动化任务编排。

### 2.4 约束与边界
- **桌面端架构**：使用 Tauri + Rust，不以 Java 作为核心开发语言。
- **安全性**：高风险操作必须二次确认并记录审计日志。
- **可解释性**：所有脚本执行前必须展示说明、风险与影响范围。

## 3. UI 与视觉设计方案（Designer 视角）

### 3.1 视觉风格定位
- **风格**：现代、专业、极简、高对比。
- **主题**：深色为主，强调色用于关键数据。
- **交互**：微动效（Hover 高亮、状态角标）。

- **右侧辅助栏**：异常告警、AI 建议、快速操作。

1. **首页仪表盘**：资源概览 + 异常提示 + AI 推荐。
2. **进程管理**：进程树 + 资源条 + 快捷操作区。
3. **端口管理**：监听列表 + 连接详情 + 一键处理。
4. **工具箱**：脚本卡片 + 说明 + 执行日志。
5. **AI 助手**：对话 + 行为建议 + 风险说明。

## 4. 技术架构与实现方案（架构师/开发专家视角）

### 4.1 总体架构（Tauri + Rust + Web 前端）

```
┌──────────────────────────────┐
│          UI Frontend          │
│  React/Vue + Tailwind/Fluent   │
└─────────────┬────────────────┘
              │ Tauri Bridge
┌─────────────▼────────────────┐
│        Rust Core Service      │
│  系统采集 / 脚本执行 / 权限控制 │
└─────────────┬────────────────┘
              │ Windows APIs
┌─────────────▼────────────────┐
│     WMI / ETW / WinAPI         │
└──────────────────────────────┘
```

### 4.2 技术选型
- **桌面框架**：Tauri（轻量、安全、性能优秀）。
- **核心能力**：Rust 调用 Windows API（WMI、ETW、Performance Counters）。
- **前端**：React 或 Vue + Fluent UI / Ant Design + Tailwind。
- **AI 层**：
  - 本地模型（轻量推理） + 云端 API（复杂推理）。
  - 风险控制、隐私权限、提示可解释。
- **非 Java 核心原则**：核心采集与执行引擎保持 Rust 实现。

### 4.3 技术规范（建议约定）
- **数据采样**：默认 1s 采样，支持用户调整；资源曲线保留 24h 滚动窗口。
- **权限管理**：管理员权限仅用于需要的操作，且必须弹窗提示。
- **日志规范**：分为操作日志、系统采样日志、执行结果日志。
- **脚本规范**：脚本包含名称、描述、参数、风险等级、回滚建议、执行输出。
- **前后端通信**：使用 Tauri Command + 事件推送，避免阻塞 UI。

### 4.4 模块实现细节

#### 资源监控
- 定时采样 Windows Performance Counters。
- 进程级别指标：CPU/内存/磁盘/网络。
- 前端图表：趋势图 + 资源条。

#### 进程/服务管理
- 使用 WinAPI 枚举进程/服务。
- 获取签名、路径、权限等元数据。
- 操作类功能需管理员权限与二次确认。

#### 网络端口
- Rust 结合 netstat 等系统接口 + WMI 关联进程。
- 解析端口 -> PID -> 应用详情。
- 提供端口释放与防火墙规则封装。

#### 脚本与工具箱
- 定义脚本模板结构（名称 / 参数 / 风险 / 说明）。
- 统一执行服务：输出日志、执行状态、失败处理。

#### AI 助手
- 系统状态摘要 -> AI 解释 -> 建议生成。
- 自然语言 -> 脚本 -> 可视化确认。

## 6. 设计原则
- **安全第一**：危险操作必须二次确认。
- **可解释性**：执行前说明原因、风险、后果。
- **可追溯性**：所有操作可查看日志与回滚建议。
- **扩展性**：以插件与脚本体系持续演进。

## 7. 项目愿景

Win-Top 将成为 Windows 端最专业、最智能、最可扩展的资源管理与系统维护工具。它不仅是一款监控工具，更是一个“系统专家”，帮助用户理解电脑、管理电脑、优化电脑。

## 8. 执行计划（开始实施）

> 目标：以 Tauri + Rust 为核心，先完成可运行的 MVP，再逐步扩展能力。

### 8.1 第一阶段（MVP 研发）
1. **基础工程搭建**：Tauri 初始化、前端框架选型、基础布局与导航。
2. **资源监控原型**：CPU/内存/磁盘/网络四大面板 + 简易图表。
3. **进程与端口基础**：进程列表、端口占用查询与关联 PID。
4. **脚本工具箱**：内置 10 个常用脚本卡片，可一键执行。
5. **AI 基础问答**：引入 AI API 进行状态解读与建议生成。

### 8.2 第二阶段（功能增强）
- 异常检测、日志审计、脚本参数化 UI。
- 磁盘清理与优化建议。
- 工具箱脚本模板编辑器。

### 8.3 第三阶段（生态扩展）
- 插件系统、脚本市场。
- 多设备监控与远程管理。
- AI 自动优化建议与可视化任务编排。
